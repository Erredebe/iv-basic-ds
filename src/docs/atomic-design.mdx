# Atomic Design

IV Basic DS organiza sus componentes con Atomic Design para decidir alcance, composicion y documentacion antes de implementar.

La taxonomia es interna del sistema. La API publica mantiene tags estables con prefijo `iv-`, por ejemplo `iv-button` e `iv-dialog`.

## Niveles

- Foundations: tokens, estilos globales y reglas transversales. No son componentes Stencil.
- Atoms: piezas minimas e indivisibles de interfaz, como acciones, inputs o indicadores simples.
- Molecules: composiciones pequenas de atoms o elementos nativos con comportamiento propio.
- Organisms: bloques complejos que agrupan molecules y atoms para resolver una region de UI.
- Templates: estructuras de pagina sin contenido final, pensadas como patrones de layout.
- Pages: ejemplos o pantallas completas para validar composicion; viven en demos o documentacion.

## Componentes Actuales

- Atoms: `iv-button`, `iv-icon`, `iv-input`.
- Molecules: `iv-dialog`.
- Organisms: sin componentes publicados todavia.
- Templates: sin componentes publicados todavia.

## Reglas

- Cada componente nuevo debe declarar su nivel atomo/molecula/organismo/template antes de implementarse.
- El nivel atomico define la carpeta fuente: `src/components/<level>/<component>/`.
- Los tags publicos no incluyen el nivel: se usa `iv-button`, no `iv-atom-button`.
- Storybook refleja el nivel en el titulo: `Atoms/Button`, `Molecules/Dialog`, etc.
- Las demos HTML siguen siendo paginas de validacion separadas por nivel en `/demos/atoms/*.html`, `/demos/molecules/*.html`, etc.; no son componentes de nivel `pages`.
