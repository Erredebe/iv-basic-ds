# Accesibilidad

El DS prioriza elementos HTML nativos y evita Shadow DOM por defecto.

## Button

- `iv-button` renderiza un `<button>` nativo o un `<a>` nativo si recibe `href`.
- Usa `type="button"` por defecto para evitar submits accidentales.
- Soporta props semanticas reflejadas (`label`, `labelled-by`, `described-by`, `controls`, `expanded`, `pressed`, `has-popup` y `current`) y las aplica como ARIA solo al `<button>` o `<a>` interno.
- No usar `aria-*` como API publica del host cuando el componente renderiza un control nativo interno, para evitar semantica duplicada.
- Si se usa como enlace con `target="_blank"`, el contenido debe avisar al usuario de que abre una nueva pestana.

## Dialog

- `iv-dialog` envuelve un `<dialog>` nativo.
- Registra `dialog-polyfill` si el navegador no soporta `showModal()`.
- Usar `dialog-role="alertdialog"` solo para confirmaciones criticas.
- En `iv-dialog`, preferir `labelled-by` cuando hay titulo visible; el componente lo aplica como `aria-labelledby` al `<dialog>` nativo interno.
- En `iv-dialog`, usar `label` solo si no hay titulo visible; el componente lo aplica como `aria-label` al `<dialog>` nativo interno.
- En `iv-dialog`, omitir `described-by` cuando el contenido es complejo, multiparrafo o estructurado.
- `initial-focus` y `restore-focus` son opt-in para evitar falsos focos en mobile screen readers.

## Verificacion Recomendada

- Chrome + NVDA.
- Firefox + NVDA.
- Safari macOS + VoiceOver.
- Safari iOS + VoiceOver.
- Android Chrome + TalkBack.
